Why Are We Doing This?

Transformers like BERT are incredibly powerful, but their massive size makes them slow, memory-heavy, and expensive to deploy in real-world production environments. The goal of model compression is to shrink a model's physical file size and drastically speed up inference latency (how fast it makes predictions) without lobotomizing its accuracy.

To test this, I built a complete three-stage compression architecture applied to DistilBERT for news topic classification using the AG News dataset. By starting with a model that has already been distilled (DistilBERT), we can push the absolute boundaries of how small and fast a network can get before its structural integrity fails.

Step 2: What is Pruning?

What is it? Pruning is the physical removal of the least important parts of a neural network to reduce computation. Think of it like trimming the dead branches off a tree.

In this architecture, we use two types of pruning:

  • Attention Head Pruning: Not all attention heads contribute equally. We calculate an importance score for each head and completely remove the bottom 60%.
  • Magnitude Pruning: We zero out the smallest, least impactful weights in the Feed-Forward Network (FFN). This creates a combined 31.1% overall sparsity across 58.6M parameters.
# 1. Attention Head Pruning
head_importance = compute_head_importance(model_pruned, small_loader, n_batches=20)
model_pruned, heads_pruned = prune_heads_by_importance(model_pruned, head_importance, prune_ratio=0.6)

# 2. Magnitude Pruning on FFN and Output Layers
import torch.nn.utils.prune as prune

for name, module in model_pruned.named_modules():
    if isinstance(module, torch.nn.Linear):
        if any(x in name for x in ["ffn.lin1", "ffn.lin2", "attention.out_lin", "classifier"]):
            prune.l1_unstructured(module, name="weight", amount=0.6)
            prune.remove(module, "weight")

Step 3: What is Knowledge Distillation (KD) and Why Use It?

What is it? Knowledge Distillation is a technique where a smaller, compressed model (the "Student") is trained to mimic the behavior of a larger, highly accurate model (the "Teacher").

Why use it? When you aggressively prune a model by ripping out attention heads and zeroing weights, it suffers a drop in accuracy. KD helps the student recover this lost performance. Instead of learning purely from binary labels (like 1 for Sports, 0 for everything else), the student learns from the Teacher's "soft probabilities" (e.g., 85% sure it's Sports, 15% sure it's Business). This richer, nuanced signal helps the sparse model regain its predictive power.

import torch.nn.functional as F

def distillation_loss(student_logits, teacher_logits, labels, temperature=4.0, alpha=0.7):
    # KL Divergence loss aligns student with teacher's soft outputs
    kd_loss = F.kl_div(
        F.log_softmax(student_logits / temperature, dim=1),
        F.softmax(teacher_logits / temperature, dim=1),
        reduction="batchmean"
    ) * (temperature ** 2)
    
    # Standard Cross-Entropy loss
    ce_loss = F.cross_entropy(student_logits, labels)
    
    return alpha * kd_loss + (1 - alpha) * ce_loss

Step 4: What is Quantization and Why Do We Need It?

What is it? Quantization is the process of converting the high-precision numbers used in a neural network (typically 32-bit floating-point numbers, or FP32) into lower-precision formats (like 8-bit integers, or INT8).

Why use it? An INT8 number takes up 4x less memory than an FP32 number. By dynamically quantizing our pruned model, we instantly slash its physical file size and make it run significantly faster on standard CPUs, as it requires less memory bandwidth and computational power to process.

# Applying dynamic quantization to target CPU execution
from copy import deepcopy

model_quantized = deepcopy(model_pruned).cpu()
model_quantized = torch.ao.quantization.quantize_dynamic(
    model_quantized, 
    {torch.nn.Linear}, 
    dtype=torch.qint8
)

Conclusion & Limitations

Combining structured attention pruning, knowledge distillation, and dynamic quantization produces compounding benefits. The fully compressed model achieved a 2.1× compression ratio and 63% faster CPU inference, while retaining roughly 96% of the baseline accuracy.

It is worth noting that working in environments like Google Colab presents specific limitations. Calculating head importance and executing magnitude pruning requires heavy tensor operations, meaning memory management is crucial to avoid Out-Of-Memory (OOM) GPU crashes. Additionally, because dynamically quantized models natively target CPU execution, GPU testing is constrained at that stage.
